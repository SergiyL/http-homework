	General:
	Request URL: url адреса сторінки
	Request Method:   тип http запиту
	Status Code:  код стану http
	Remote Address:   ip адреса ресурсу
	Referrer Policy:   вказує на те яка інформація повинна бути включена у запити

	Response Headers:
	accept-ranges:  маркер що використовується сервером для оголошення підтримки часткових запитів
	age:   кількість секунд з моменту модифікації ресурсу
	alt-svc:  повідомляє клієнту про наявність альтернативного сервісу 
	backend-timing:   
	cache-control:  основні директиви для керування кешуванням 
	connection:  дані про проведенні з'єднання
	content-encoding:  спосіб кодування контенту при передачі
	content-language:  один або кілька природніх мов вмісту контенту
	content-length:  розмір вмісту контенту в октетах (зазвичай їх називають байтами)
	content-security-policy:  додаткова властивість безпеки що допомагає визначати та зменшувати різні типи атак
	content-type:   формат і спосіб представлення ресурсу
	date:  дата генерації відгуку
	expect-ct:  заголовок що дозволяє сайту надавти звіт про дотримання вимог прозорості сертифікатів
	expires:  дата закінчення терміну актуальності контенту
	last-modified:  дата останньої модифікації
	link:  вказує на логічно зв'язаний з сутністю ресурс аналогічно тегу <LINK> в HTML
	p3p:  протокол що дозволяє сайтам інформувати браузер про ймовірне отримання особистих даних користувача
	pragma:  застарілий хедер для http/1.0 аналог cache-control: 
	server:  список назв і версій веб-серверу і його компонентів з коментарями
	set-cookie:  використовується для надсилання кукі з серверу до агента користувача
	status:  код стану http
	strict-transport-security:  вказує браузеру на потребу використання виключно https замість http
	transfer-encoding:  список способів кодування, котрі були використані до повідомлення для передачі
	vary:  список полів з запиту, що описують ресурс, і котрі були приняті до уваги
	via:  список версій протоколу, назв і версій проксі-серверів, через котрих пройшло повідомлення
	x-content-type-options:  маркер що використовується сервером для вказівки що стандартизований спосіб індикації (MIME) не повинен змінюватись 
	x-fb-debug:  специфічний заголовок фейсбука для надсилання баг репортів
	x-frame-options:  вказує чи повинен браузер відображати сторінки з тегами <frame>, <iframe> або <object> 
	x-ua-compatible:  керує режимом відображення сторінок в браузерах ІЕ8+
	x-varnish:  надає можливість знайти правильні логи транзакцій
	x-xss-protection:  особливість браузера зупиняти завантаження сторінки при виявленні XSS атаки

	Request Headers:
	:authority:  псевдо хедер що використовується http/2 вказує на адресу для запитів
	:method:  тип http запиту
	:path:  серверний шлях до файлу сторінки
	accept:  список допустимих форматів ресурсу
	accept-encoding:  перелік підтримуваних способів кодування вмісту сутності при передачі
	accept-language:  список підтримуваних природніх мов
	cookie:  містить попередньо надіслані сервером кукі разом з set-cookie хедером 
	referer:  URI ресурсу, після котрого клієнт зробив поточний запит
	upgrade-insecure-requests:  вказує на необхідність вткористання безпечного з'єднання
	user-agent:  список назв і версій клієнту і його компонентів з коментарями
