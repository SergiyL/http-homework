1. Основні відмінності HTTP/1.1 в порівнянні з HTTP/1.0:
    - підтримує зв'язок після відповіді сервера
    - дозволяє клієнту надсилати повторні запити без очікування на відповідь попереднього запиту
    - дозволяє відправнику розбивати повідомлення на частини 
    - має безпечну захищену автентифікацію
2. Метод GET використовується для запиту даних. Метод POST для надсилання даних на сервер.  
    GET / HTTP/1.1  - зразок GET
    Host: www.url.com

    POST / HTTP/1.1 - зразок методу POST (https://developer.mozilla.org/ru/docs/Web/HTTP/Methods/POST)
    Host: foo.com
    Content-Type: application/x-www-form-urlencoded
    Content-Length: 13

    say=Hi&to=Mom

    Відмінність у запитах поміж методами полягає в тому що GET містить тільки запит та адресу джерела, а пост також повідомлення яке має бути надіслано

    3. Метод POST використвується в тих сайтах де отримувач даних може бути і їх відправником. Для прикладу www.facebook.com метод POST використовується при аторизації, при надсиланні повідомлень, розміщенні постів. www.telegra.ph при розміщенні публікацій.
